Tensor-product discretizations

As you learned when starting double integration in vector calculus, the simplest extension of an interval to two dimensions is a rectangle. We will use a particular notation for rectangles.

Definition 13.1.1 (Tensor-product domain)

A tensor-product domain is two dimensions is the rectangle

[a,b]×[c,d]={(x,y)∈R2:a≤x≤b,  c≤y≤d}. [a,b] \times [c,d] = \bigl\{ (x,y)\in\mathbb{R}^2 : a\le x \le b,\; c\le y \le d \bigr\}.[a,b]×[c,d]={(x,y)∈R2:a≤x≤b,c≤y≤d}.
(13.1.1)
Note

The × notation in (13.1.1) is called a tensor product.

The idea of the tensor product is that each variable independently varies over a fixed set. When the interval is the same in each dimension, we may write [a,b]2[a,b]^2[a,b]2.

The discretization of a two-dimensional tensor-product domain is straightforward.

Definition 13.1.2 (Tensor-product grid)

Given discretizations of two intervals,

a=x0<x1<⋯<xm=b,c=y0<y1<⋯<yn=d,a= x_0< x_1 < \cdots < x_m = b, \qquad c = y_0 < y_1 < \cdots < y_n = d,a=x0​<x1​<⋯<xm​=b,c=y0​<y1​<⋯<yn​=d,
(13.1.2)

then a tensor-product grid on [a,b]×[c,d][a,b]\times[c,d][a,b]×[c,d] is the set

{(xi,yj):i=0,…,m,  j=0,…,n}. \bigl\{ (x_i, y_j): i=0,\ldots,m,\; j=0,\ldots,n \bigr\}.{(xi​,yj​):i=0,…,m,j=0,…,n}.
(13.1.3)

Figure 13.1.1 shows a tensor-product grid on a rectangle. The grid is constructed from discretizations of xxx with m=3m=3m=3 and yyy with n=5n=5n=5. Each grid point (xi,yj)(x_i, y_j)(xi​,yj​) lies at the intersection of a vertical line through xix_ixi​ and a horizontal line through yjy_jyj​.

13.1.1Functions on grids¶

The double indexing of the grid set (13.1.3) implies an irresistible connection to matrices. Corresponding to any function f(x,y)f(x,y)f(x,y) defined on the rectangle is an (m+1)×(n+1)(m+1)\times(n+1)(m+1)×(n+1) matrix F\mathbf{F}F defined by collecting the values of fff at the points in the grid. This transformation of a function to a matrix is so important that we give it a formal name

Definition 13.1.3 (Function-to-matrix map)

The function mtx maps a function f(x,y)f(x,y)f(x,y) to an (m+1)×(n+1)(m+1)\times (n+1)(m+1)×(n+1) matrix F\mathbf{F}F by

F=mtx⁡(f)=[f(xi,yj)]i=0,…,mj=0,…,n,\mathbf{F} = \mtx(f) = \Bigl[f(x_i,y_j)\Bigr]_{\substack{i=0,\ldots,m\\j=0,\ldots,n}},F=mtx(f)=[f(xi​,yj​)]i=0,…,mj=0,…,n​​,
(13.1.4)

where the evaluations are on the grid (13.1.3).

Warning

Traditionally, the first dimension of a matrix varies in the vertical direction, while the first space coordinate xxx varies horizontally. This clash has long been a source of confusion when coding, and each language or package has its own conventions around it. In this book, the first dimension of a matrix always corresponds to the first coordinate of the function. Sometimes, this forces us to use a transpose for making plots.

13.1.2Parameterized surfaces¶

We are not limited to rectangles by tensor products. Many regions and surfaces may be parameterized by means of x(u,v)x(u,v)x(u,v), y(u,v)y(u,v)y(u,v), and z(u,v)z(u,v)z(u,v), where uuu and vvv lie in a rectangle. Such “logically rectangular” surfaces include the unit disk,

{x=ucos⁡v,y=usin⁡v,0≤u<1,0≤v≤2π,\left\{ \begin{aligned} x &= u \cos v, \\ y &= u \sin v, \\ \end{aligned} \right. \qquad \qquad \left. \begin{aligned} 0 & \le u < 1, \\ 0 & \le v \le 2\pi, \end{aligned} \right.{xy​=ucosv,=usinv,​00​≤u<1,≤v≤2π,​
(13.1.5)

and the unit sphere,

{x=cos⁡usin⁡v,y=sin⁡usin⁡v,z=cos⁡v,0≤u<2π,0≤v≤π.\left\{ \begin{aligned} x &= \cos u \sin v,\\ y &= \sin u \sin v,\\ z &= \cos v, \end{aligned} \right. \qquad \qquad \left. \begin{aligned} 0 & \le u < 2\pi, \\ 0 &\le v \le \pi. \end{aligned} \right.⎩⎨⎧​xyz​=cosusinv,=sinusinv,=cosv,​00​≤u<2π,≤v≤π.​

13.1.3Partial derivatives¶

In order to solve boundary-value problems in one dimension by collocation, we replaced an unknown function u(x)u(x)u(x) by a vector of its values at selected nodes and discretized the derivatives in the equation using differentiation matrices. We use the same ideas in the 2D case: we represent a function by its values on a grid, and multiplication by differentiation matrices to construct discrete analogs of the partial derivatives ∂u∂x\frac{\partial u}{\partial x}∂x∂u​ and ∂u∂y\frac{\partial u}{\partial y}∂y∂u​.

Consider first ∂u∂x\frac{\partial u}{\partial x}∂x∂u​. In the definition of this partial derivative, the independent variable yyy is held constant. Note that yyy is constant within each column of U=mtx⁡(u)\mathbf{U} = \mtx(u)U=mtx(u). Thus, we may regard a single column uj\mathbf{u}_juj​ as a discretized function of xxx and, as usual, left-multiply by a differentiation matrix Dx\mathbf{D}_xDx​ such as (10.3.6). We need to do this for each column of U\mathbf{U}U by Dx\mathbf{D}_xDx​, which is accomplished by DxU\mathbf{D}_x \mathbf{U}Dx​U. Altogether,

mtx⁡(∂u∂x)≈Dx mtx⁡(u).\mtx\left( \frac{\partial u}{\partial x} \right) \approx \mathbf{D}_x \, \mtx(u).mtx(∂x∂u​)≈Dx​mtx(u).
(13.1.7)

This relation is not an equality, because the left-hand side is a discretization of the exact partial derivative, while the right-hand side is a finite-difference approximation. Yet it is a natural analog for partial differentiation when we are given not u(x,y)u(x,y)u(x,y) but only the grid value matrix U.\mathbf{U}.U.

Now we tackle ∂u∂y\frac{\partial u}{\partial y}∂y∂u​. Here the inactive coordinate xxx is held fixed within each row of U\mathbf{U}U. However, if we transpose U\mathbf{U}U, then the roles of rows and columns are swapped, and now yyy varies independently down each column. This is analogous to the situation for the xxx-derivative, so we left-multiply by a finite-difference matrix Dy\mathbf{D}_yDy​, and then transpose the entire result to restore the roles of xxx and yyy in the grid. Fortunately, linear algebra allows us to express the sequence transpose–left-multiply–transpose more compactly:

mtx⁡(∂u∂y)≈(DyUT)T=mtx⁡(u) DyT.\mtx\left( \frac{\partial u}{\partial y} \right) \approx \Bigl(\mathbf{D}_y \mathbf{U}^T\Bigr)^T = \mtx(u)\, \mathbf{D}_y^T.mtx(∂y∂u​)≈(Dy​UT)T=mtx(u)DyT​.
(13.1.8)

Keep in mind that the differentiation matrix Dx\mathbf{D}_xDx​ is based on the discretization x0,…,xmx_0,\ldots,x_mx0​,…,xm​, and as such it must be (m+1)×(m+1)(m+1)\times (m+1)(m+1)×(m+1). On the other hand, Dy\mathbf{D}_yDy​ is based on y0,…,yny_0,\ldots,y_ny0​,…,yn​ and is (n+1)×(n+1)(n+1)\times (n+1)(n+1)×(n+1). This is exactly what is needed dimensionally to make the products in (13.1.7) and (13.1.8) consistent. More subtly, if the differentiation is based on equispaced grids in each variable, the value of hhh in a formula such as (5.4.5) will be different for Dx\mathbf{D}_xDx​ and Dy\mathbf{D}_yDy​.

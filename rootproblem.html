The rootfinding problem

For the time being we will focus on the rootfinding problem for single functions of one variable.

Definition 4.1.1 (Rootfinding problem)

Given a continuous scalar function fff of a scalar variable, find a real number rrr, called a root, such that f(r)=0f(r)=0f(r)=0.

The formulation f(x)=0f(x)=0f(x)=0 is general enough to solve any equation. If we are trying to solve an equation g(x)=h(x)g(x)=h(x)g(x)=h(x), we can define f=g−hf=g-hf=g−h and find a root of fff.

Unlike the linear problems of the earlier chapters, the usual situation here is that the root cannot be produced in a finite number of operations, even in exact arithmetic. Instead, we seek a sequence of approximations that formally converge to the root, stopping when some member of the sequence seems to be good enough, in a sense we will clarify later.

Example 4.1.1 (The rootfinding problem for Bessel functions)

In the theory of vibrations of a circular drum, the displacement of the drumhead can be expressed in terms of pure harmonic modes,

Jm(ωk,mr)cos⁡(mθ)cos⁡(cωk,mt),J_m(\omega_{k,m} r) \cos(m\theta) \cos(c \omega_{k,m} t),Jm​(ωk,m​r)cos(mθ)cos(cωk,m​t),
(4.1.1)

where (r,θ)(r,\theta)(r,θ) are polar coordinates, 0≤r≤10\le r\le 10≤r≤1, ttt is time, mmm is a positive integer, ccc is a material parameter, and JmJ_mJm​ is a Bessel function of the first kind. The quantity ωk,m\omega_{k,m}ωk,m​ is a resonant frequency and is a positive root of the equation

Jm(ωk,m)=0,J_m(\omega_{k,m}) = 0,Jm​(ωk,m​)=0,
(4.1.2)

which states that the drumhead is clamped around the rim. Bessel functions often appear in physical problems featuring radial symmetry, and tabulating approximations to the zeros of Bessel functions occupied numerous mathematician-hours before computers were on the scene.

4.1.1Conditioning, error, and residual¶

In the rootfinding problem, the data is a continuous function fff and the result is a root. (This overrides our Chapter 1 notation of fff as the map from data to result.) How does the result change in response to perturbations in fff? We will compute an absolute condition number rather than a relative one.

You might wonder about the relevance of perturbing a function as data to a problem. If nothing else, the values of fff will be represented in floating point and thus subject to rounding error. Furthermore, in many applications, fff might not be a simple formula but the result of a computation that uses an inexact algorithm. While there are infinitely many possible perturbations to a function, a constant perturbation is enough to get the main idea.

We assume fff has at least one continuous derivative near a particular root rrr. Suppose that fff is perturbed to f~(x)=f(x)+ϵ\tilde{f}(x) = f(x) + \epsilonf~​(x)=f(x)+ϵ. As a result, the root (if it still exists) will be perturbed to r~=r+δ\tilde{r} = r + \deltar~=r+δ such that f~(r~)=0\tilde{f}(\tilde{r})=0f~​(r~)=0. We now compute an absolute condition number κr\kappa_rκr​, which is the ratio ∣δϵ∣\left | \frac{\delta}{\epsilon} \right|∣∣​ϵδ​∣∣​ as ϵ→0\epsilon\to 0ϵ→0.

Using Taylor’s theorem,

0=f(r+δ)+ϵ≈f(r)+f′(r)δ+ϵ. 0 = f(r+\delta) + \epsilon \approx f(r) + f'(r) \delta + \epsilon.0=f(r+δ)+ϵ≈f(r)+f′(r)δ+ϵ.
(4.1.3)

Since rrr is a root, we have f(r)=0f(r)=0f(r)=0. This lets us relate δ to ε, and their ratio is the condition number.

Theorem 4.1.1 (Condition number of rootfinding)

If fff is differentiable at a root rrr, then the absolute condition number of rrr with respect to constant changes in fff is

κr=∣f′(r)∣−1. \kappa_r = \bigl| f'(r) \bigr|^{-1}.κr​=∣∣​f′(r)∣∣​−1.
(4.1.4)

We say κr=∞\kappa_r = \inftyκr​=∞ if f′(r)=0f'(r)=0f′(r)=0.

Equivalently, (4.1.4) is just the magnitude of the derivative of the inverse function f−1f^{-1}f−1 at zero.

We must accept that when ∣f′∣|f'|∣f′∣ is small at the root, it may not be possible to get a small error in a computed root estimate. As always, the error is not a quantity we can compute without knowing the exact answer. There is something else we can measure, though.

Definition 4.1.2 (Rootfinding residual)

If r~\tilde{r}r~ approximates a root rrr of function fff, then the residual at r~\tilde{r}r~ is f(r~)f(\tilde{r})f(r~).

It stands to reason that a small residual might be associated with a small error. To quantify the relationship, let r~\tilde{r}r~ approximate root rrr, and define the new function g(x)=f(x)−f(r~)g(x)=f(x)-f(\tilde{r})g(x)=f(x)−f(r~). Trivially, g(r~)=0g(\tilde{r})=0g(r~)=0, meaning that r~\tilde{r}r~ is a true root of ggg. Since the difference g(x)−f(x)g(x)-f(x)g(x)−f(x) is the residual value f(r~)f(\tilde{r})f(r~), the residual is the distance to an exactly solved rootfinding problem.

Observation 4.1.1

The backward error in a root estimate is equal to the residual.

In general, it is not realistic to expect a small error in a root approximation if the condition number (4.1.4) is large. However, we can gauge the backward error from the residual.

4.1.2Multiple roots¶

The condition number (4.1.4) naturally leads to the question of what happens if f′(r)=0f'(r)=0f′(r)=0 at a root rrr. The following definition agrees with and extends the notion of algebraic multiplicity in a polynomial to roots of more general differentiable functions.

Definition 4.1.3 (Multiplicity of a root)

If f(r)=f′(r)=⋯=f(m−1)(r)=0f(r)=f'(r)=\cdots=f^{(m-1)}(r)=0f(r)=f′(r)=⋯=f(m−1)(r)=0, but f(m)(r)≠0f^{(m)}(r)\neq 0f(m)(r)=0, then we say fff has a root of multiplicity mmm at rrr. In particular, if f(r)=0f(r)=0f(r)=0 and f′(r)≠0f'(r)\neq 0f′(r)=0, then m=1m=1m=1 and we call rrr a simple root.

Another useful characterization of multiplicity mmm is that f(x)=(x−r)mq(x)f(x)=(x-r)^m q(x)f(x)=(x−r)mq(x) for a differentiable qqq with q(r)≠0q(r)\neq 0q(r)=0.

When rrr is a nonsimple root, the condition number (4.1.4) is effectively infinite.[1] However, even if rrr is simple, we should expect difficulty in rootfinding if the condition number is very large. This occurs when ∣f′(r)∣|f'(r)|∣f′(r)∣ is very small, which means that the quotient qqq satisfies q(r)≈0q(r)\approx 0q(r)≈0 and another root of fff is very close to rrr. We made the same observation about polynomial roots all the way back in Demo 1.4.3.

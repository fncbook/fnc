Fixed-point iteration

In this section, we consider the alternative form of the rootfinding problem known as the fixed-point problem.

Definition 4.2.1 (Fixed-point problem)

Given a function ggg, the fixed-point problem is to find a value ppp, called a fixed point, such that g(p)=pg(p)=pg(p)=p.

Given fff for rootfinding, we could define g(x)=x−f(x)g(x)=x-f(x)g(x)=x−f(x), and then f(r)=0f(r)=0f(r)=0 implies g(r)=rg(r)=rg(r)=r and vice versa. There are infinitely many ways to make this transformation, such as g(x)=x+cf(x)g(x)=x+cf(x)g(x)=x+cf(x) for any constant ccc. The process can be reversed, too. Given g(x)g(x)g(x), we could define f(x)=x−g(x)f(x)=x-g(x)f(x)=x−g(x), and then g(p)=pg(p)=pg(p)=p implies f(p)=0f(p)=0f(p)=0.

There is an extraordinarily simple way to try to find a fixed point of any given g(x)g(x)g(x).

Algorithm 4.2.1 (Fixed-point iteration)

Given function ggg and initial value x1x_1x1​, define

xk+1=g(xk),k=1,2,…. x_{k+1} = g(x_k), \qquad k=1,2,\ldots.xk+1​=g(xk​),k=1,2,….
(4.2.1)

This is our first example of an iterative algorithm that never quite gets to the answer, even if we use exact numbers. The idea is to generate a sequence of values that one hopes will converge to the correct result, and stop when we are satisfied that we are close enough to the limit.

4.2.1Series analysis¶

In Demo 4.2.1, the two computed iterations differ only in the choice of x1x_1x1​. In the first case we evidently generated a sequence that converged to one of the fixed points. In the second case, however, the generated sequence diverged.[1] The easiest way to uncover the essential difference between the two cases is to use a Taylor series expansion.

Suppose a fixed point ppp is the desired limit of an iteration x1,x2,…x_1,x_2,\ldotsx1​,x2​,…. It’s often easier to express quantities in terms of the error sequence ϵ1,ϵ2,…,\epsilon_1,\epsilon_2,\ldots,ϵ1​,ϵ2​,…, where ϵk=xk−p\epsilon_k=x_k-pϵk​=xk​−p. Starting from (4.2.1), we have

ϵk+1+p=g(ϵk+p)=g(p)+g′(p)ϵk+12g′′(p)ϵk2+⋯ ,\begin{split} \epsilon_{k+1}+p = g( \epsilon_{k}+p ) = g(p) + g'(p) \epsilon_k + \frac{1}{2}g''(p) \epsilon_k^2 + \cdots, \end{split}ϵk+1​+p=g(ϵk​+p)=g(p)+g′(p)ϵk​+21​g′′(p)ϵk2​+⋯,​
(4.2.2)

assuming that ggg has at least two continuous derivatives. But by definition, g(p)=pg(p)=pg(p)=p, so

ϵk+1=g′(p)ϵk+O(ϵk2). \epsilon_{k+1} = g'(p) \epsilon_k + O(\epsilon_k^2).ϵk+1​=g′(p)ϵk​+O(ϵk2​).
(4.2.3)

If the iteration is to converge to ppp, the errors must approach zero. In this case we can neglect the second-order term and conclude that ϵk+1≈g′(p)ϵk\epsilon_{k+1} \approx g'(p) \epsilon_kϵk+1​≈g′(p)ϵk​. This is consistent with convergence if ∣g′(p)∣<1|g'(p)|<1∣g′(p)∣<1. However, if ∣g′(p)∣>1|g'(p)| >1∣g′(p)∣>1, we are led to the conclusion that the errors must grow, not vanish, even if they start quite small.

Observation 4.2.1

fixed-point iteration for a differentiable g(x)g(x)g(x) converges to a fixed point ppp if the initial error is sufficiently small and ∣g′(p)∣<1|g'(p)|< 1∣g′(p)∣<1. The iteration diverges for all initial values if ∣g′(p)∣>1|g'(p)| > 1∣g′(p)∣>1.

Example 4.2.2

The role of g′(p)g'(p)g′(p) is clear in Demo 4.2.1. We have g(x)=−x2+5x−3.5g(x) = -x^2+5x-3.5g(x)=−x2+5x−3.5 and g′(x)=−2x+5g'(x)=-2x+5g′(x)=−2x+5. For the first fixed point, near 2.71, we get g′(p)≈−0.42g'(p)\approx-0.42g′(p)≈−0.42, indicating convergence. For the second fixed point, near 1.29, we get g′(p)≈2.42g'(p)\approx 2.42g′(p)≈2.42, which is consistent with the observed divergence.

4.2.2Linear convergence¶

In computation we usually want to know not just whether an iteration converges but also the rate at which convergence occurs, i.e., how quickly the errors approach zero. Other things being equal, faster convergence is preferred to slower convergence, as it usually implies that the computation will take less time to achieve a desired accuracy.

The prediction of the series analysis above is that if the fixed-point iteraion converges, the errors approximately satisfy ∣ϵk+1∣=σ∣ϵk∣|\epsilon_{k+1}| = \sigma|\epsilon_k|∣ϵk+1​∣=σ∣ϵk​∣, for σ=∣g′(p)∣<1\sigma = |g'(p)| < 1σ=∣g′(p)∣<1. This is a well-known type of convergence.

Definition 4.2.2 (Linear convergence)

Suppose a sequence xkx_kxk​ approaches limit x∗x^*x∗. If the error sequence ϵk=xk−x∗\epsilon_k=x_k - x^*ϵk​=xk​−x∗ satisfies

lim⁡k→∞∣ϵk+1∣∣ϵk∣=σ<1, \lim_{k\to\infty} \frac{|\epsilon_{k+1}|}{|\epsilon_k|} = \sigma < 1,k→∞lim​∣ϵk​∣∣ϵk+1​∣​=σ<1,
(4.2.4)

then the sequence displays linear convergence. The number σ is called the convergence rate.

If we suppose that the ratios in (4.2.4) all equal σ (i.e., perfect linear convergence), then ∣ϵk∣=Cσk|\epsilon_k| = C \sigma^k∣ϵk​∣=Cσk. Taking logs, we get

log⁡∣ϵk∣=k(log⁡σ)+(log⁡C). \log |\epsilon_k| = k(\log \sigma) + (\log C).log∣ϵk​∣=k(logσ)+(logC).
(4.2.5)

This is in the form log⁡∣ϵk∣=αk+β\log |\epsilon_k| = \alpha k + \betalog∣ϵk​∣=αk+β, which is a linear relationship.

Observation 4.2.2 (Linear convergence in practice)

Linear convergence is marked by an approximate reduction of the error at each iteration by a constant factor, the convergence rate σ. When graphed on a log-linear scale, the errors lie on a straight line whose slope is the log of the convergence rate. Both phenomena manifest most strongly at the latest iterations.

4.2.3Contraction maps¶

The convergence condition σ=∣g′(p)∣<1\sigma=|g'(p)|<1σ=∣g′(p)∣<1 derived by series expansion is a special case of a more general condition.

Definition 4.2.3

A function ggg is said to satisfy a Lipschitz condition with constant LLL on the interval S⊂RS\subset\mathbb{R}S⊂R if, for all s,t∈Ss,t\in Ss,t∈S,

∣g(s)−g(t)∣≤L∣s−t∣. \bigl| g(s)-g(t) \bigr| \le L \bigl| s-t \bigr|.∣∣​g(s)−g(t)∣∣​≤L∣∣​s−t∣∣​.
(4.2.6)

It can be shown that a function satisfying (4.2.6) is continuous in SSS. If L<1L<1L<1, we call ggg a contraction mapping because distances between points all decrease after an application of ggg. This situation leads to a major result about fixed points.

Theorem 4.2.1 (Contraction mapping)

Suppose that ggg satisfies (4.2.6) with L<1L<1L<1 on an interval SSS. Then SSS contains exactly one fixed point ppp of ggg. If x1,x2,…x_1,x_2,\ldotsx1​,x2​,… are generated by the fixed-point iteraion (4.2.1), and x1,x2,…x_1,x_2,\ldotsx1​,x2​,… all lie in SSS, then ∣xk−p∣≤Lk−1∣x1−p∣|x_k-p|\le L^{k-1} |x_1-p|∣xk​−p∣≤Lk−1∣x1​−p∣ for all k>1k>1k>1.

Proof

(partial proof) First we show there is at most one fixed point in SSS. Suppose g(t)=tg(t)=tg(t)=t and g(s)=sg(s)=sg(s)=s in SSS. Then by (4.2.6), ∣s−t∣=∣g(s)−g(t)∣≤L∣s−t∣|s-t|=|g(s)-g(t)|\le L|s-t|∣s−t∣=∣g(s)−g(t)∣≤L∣s−t∣, which for L<1L<1L<1 is possible only if ∣s−t∣=0|s-t|=0∣s−t∣=0, so s=ts=ts=t.

Now suppose that for some p∈Sp\in Sp∈S, g(p)=pg(p)=pg(p)=p. By the definition of the fixed-point iteraion and the Lipschitz condition,

∣xk+1−p∣=∣g(xk)−g(p)∣≤L∣xk−p∣,|x_{k+1} - p | = |g(x_k) - g(p)| \le L |x_k-p|,∣xk+1​−p∣=∣g(xk​)−g(p)∣≤L∣xk​−p∣,

which shows that xk→px_k\to pxk​→p as k→∞k\to \inftyk→∞. To show that ppp must exist and complete the proof, one needs to apply the Cauchy theory of convergence of a sequence, which is beyond the scope of this text.

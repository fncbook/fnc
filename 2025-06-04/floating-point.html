Floating-point numbers

The real number set R\realR is infinite in two ways: it is unbounded and continuous. In most practical computing, the latter kind of infiniteness is much more consequential than the former, so we turn our attention there first.

Definition 1.1.1 (Floating-point numbers)

The set F\floatF of floating-point numbers consists of zero and all numbers of the form

±(1+f)×2n, \pm (1 + f) \times 2^n,±(1+f)×2n,
(1.1.1)

where nnn is an integer called the exponent, and 1+f1+f1+f is the significand, in which

f=∑i=1dbi 2−i,bi∈{0,1}, f = \sum_{i=1}^d b_i \, 2^{-i}, \qquad b_i\in\{0,1\},f=i=1∑d​bi​2−i,bi​∈{0,1},
(1.1.2)

for a fixed integer ddd called the binary precision.

Equation (1.1.2) represents the significand as a number in [1,2)[1,2)[1,2) in base-2 form. Equivalently,

f=2−d ∑i=1dbi 2d−i=2−dzf = 2^{-d}\, \sum_{i=1}^{d} b_{i} \, 2^{d-i} = 2^{-d} zf=2−di=1∑d​bi​2d−i=2−dz
(1.1.3)

for an integer zzz in the set {0,1,…,2d−1}\{0,1,\ldots,2^d-1\}{0,1,…,2d−1}. Consequently, starting at 2n2^n2n and ending just before 2n+12^{n+1}2n+1 there are exactly 2d2^d2d evenly spaced numbers belonging to F\floatF.

Example 1.1.1

Suppose d=2d=2d=2. Taking n=0n=0n=0 in (1.1.1), we enumerate

1+04, 1+14, 1+24, 1+34.1 + \frac{0}{4}, \: 1 + \frac{1}{4}, \: 1 + \frac{2}{4}, \: 1 + \frac{3}{4}.1+40​,1+41​,1+42​,1+43​.
(1.1.4)

These are the only members of F\floatF in the semi-closed interval [1,2)[1,2)[1,2), and they are separated by spacing 14\tfrac{1}{4}41​.

Taking n=1n=1n=1 doubles each of the values in the list above, and n=−1n=-1n=−1 halves them. These give the floating-point numbers in [2,4)[2,4)[2,4) and [1/2,1)[1/2,1)[1/2,1), respectively. The spacing between them also is doubled and halved, respectively.

Observe that the smallest element of F\floatF that is greater than 1 is 1+2−d1+2^{-d}1+2−d, and we call the difference machine epsilon.[1]

Definition 1.1.2 (Machine epsilon)

For a floating-point set with ddd binary digits of precision, machine epsilon (or machine precision) is ϵmach=2−d\macheps = 2^{-d}ϵmach​=2−d.

We define the rounding function fl⁡(x)\fl(x)fl(x) as the map from real number xxx to the nearest member of F\floatF. The distance between the floating-point numbers in [2n,2n+1)[2^n,2^{n+1})[2n,2n+1) is 2nϵmach=2n−d2^n\macheps=2^{n-d}2nϵmach​=2n−d. As a result, every real x∈[2n,2n+1)x \in [2^n,2^{n+1})x∈[2n,2n+1) is no farther than 2n−d−12^{n-d-1}2n−d−1 away from a member of F\floatF. Therefore, we conclude that ∣fl⁡(x)−x∣≤12(2n−d)|\fl(x)-x| \le \tfrac{1}{2}(2^{n-d})∣fl(x)−x∣≤21​(2n−d), which leads to the bound

∣fl⁡(x)−x∣∣x∣≤2n−d−12n≤12ϵmach.\frac{|\fl(x)-x|}{|x|} \le \frac{2^{n-d-1}}{2^n} \le \tfrac{1}{2}\macheps.∣x∣∣fl(x)−x∣​≤2n2n−d−1​≤21​ϵmach​.
(1.1.5)

In words, every real number is represented with a uniformly bounded relative precision. Inequality (1.1.5) holds true for negative xxx as well. In Exercise 1.1.2 you are asked to show that an equivalent statement is that

fl⁡(x)=x(1+ϵ)for some ∣ϵ∣≤12ϵmach. \fl(x)=x(1+\epsilon) \quad \text{for some $|\epsilon|\le \tfrac{1}{2}\macheps$.}fl(x)=x(1+ϵ)for some ∣ϵ∣≤21​ϵmach​.
(1.1.6)

The value of ε depends on xxx, but this dependence is not usually shown explicitly.

1.1.1Precision and accuracy¶

It may help to recast (1.1.1) and (1.1.2) in terms of base 10:

±(b0+∑i=1dbi 10−i)×10n=±(b0.b1b2⋯bd)×10n,\pm \left(b_0 + \sum_{i=1}^d b_i \, 10^{-i} \right) \times 10^n = \pm (b_0.b_1b_2\cdots b_d) \times 10^n,±(b0​+i=1∑d​bi​10−i)×10n=±(b0​.b1​b2​⋯bd​)×10n,
(1.1.7)

where each bib_ibi​ is in {0,1,…,9}\{0,1,\ldots,9\}{0,1,…,9} and b0≠0b_0\neq 0b0​=0. This is simply scientific notation with d+1d+1d+1 significant digits. For example, Planck’s constant is 6.626068×10−346.626068\times 10^{-34}6.626068×10−34 m2⋅{}^2\cdot2⋅kg/sec to seven digits. If we alter just the last digit from 8 to 9, the relative change is

0.000001×10−346.626068×10−34≈1.51×10−7.\frac{0.000001\times 10^{-34}}{6.626068\times 10^{-34}} \approx 1.51\times 10^{-7}.6.626068×10−340.000001×10−34​≈1.51×10−7.
(1.1.8)

We therefore say that the constant is given with 7 decimal digits of precision. That’s in contrast to noting that the value is given to 40 decimal places. A major advantage of floating point is that the relative precision does not depend on the choice of physical units. For instance, when expressed in eV•sec, Planck’s constant is 4.135668×10−154.135668\times 10^{-15}4.135668×10−15, which still has 7 digits but only 21 decimal places.

Floating-point precision functions the same way, except that computers prefer base 2 to base 10. The precision of a floating-point number is always ddd binary digits, implying a resolution of the real numbers according to (1.1.5).

It can be easy to confuse precision with accuracy, especially when looking at the result of a calculation on the computer. Every result is computed and represented using ddd binary digits, but not all of those digits may accurately represent an intended value. Suppose xxx is a number of interest and x~\tilde{x}x~ is an approximation to it. The absolute accuracy of x~\tilde{x}x~ is

∣x~−x∣,|\tilde{x} - x|,∣x~−x∣,
(1.1.9)

while the relative accuracy is

∣x~−x∣∣x∣.\frac{|\tilde{x} - x|}{|x|}.∣x∣∣x~−x∣​.
(1.1.10)

Absolute accuracy has the same units as xxx, while relative accuracy is dimensionless. We can also express the relative accuracy as the number of accurate digits, computed in base 10 as

−log⁡10∣x~−xx∣.-\log_{10} \left| \frac{\tilde{x}-x}{x} \right|.−log10​∣∣​xx~−x​∣∣​.
(1.1.11)

We often round this value down to an integer, but it does make sense to speak of “almost seven digits” or “ten and a half digits.”

1.1.2Double precision¶

Most numerical computing today is done in the IEEE 754 standard. This defines single precision with d=23d=23d=23 binary digits for the fractional part fff, and the more commonly used double precision with d=52d=52d=52. In double precision,

ϵmach=2−52≈2.2×10−16.\macheps = 2^{-52} \approx 2.2\times 10^{-16}.ϵmach​=2−52≈2.2×10−16.
(1.1.12)

We often speak of double-precision floating-point numbers as having about 16 decimal digits. The 52-bit significand is paired with a sign bit and 11 binary bits to represent the exponent nnn in (1.1.1), for a total of 64 binary bits per floating-point number.

Our theoretical description of F\floatF did not place limits on the exponent, but in double precision its range is limited to −1022≤n≤1023-1022\le n \le 1023−1022≤n≤1023. Thus, the largest number is just short of 21024≈2×103082^{1024}\approx 2\times 10^{308}21024≈2×10308, which is enough in most applications. Results that should be larger are said to overflow and will actually result in the value Inf. Similarly, the smallest positive number is 2−1022≈2×10−3082^{-1022}\approx 2\times 10^{-308}2−1022≈2×10−308, and smaller values are said to underflow to zero.[2]

Note the crucial difference between ϵmach=2−52\macheps=2^{-52}ϵmach​=2−52, which is the distance between 1 and the next larger double-precision number, and 2-1022, which is the smallest positive double-precision number. The former has to do with relative precision, while the latter is about absolute precision. Getting close to zero always requires a shift in thinking to absolute precision because any finite error is infinite relative to zero.

One more double-precision value is worthy of note: NaN, which stands for Not a Number. It is the result of an undefined arithmetic operation such as 0/0.

1.1.3Floating-point arithmetic¶

Computer arithmetic is performed on floating-point numbers and returns floating-point results. We assume the existence of machine-analog operations for real functions such as +++, −-−, ×, ///, \sqrt{\quad}​, and so on. Without getting into the details, we will suppose that each elementary machine operation creates a floating-point result whose relative error is bounded by ϵmach\machepsϵmach​. For example, if xxx and yyy are in F\floatF, then for machine addition ⊕\oplus⊕ we have the bound

∣(x⊕y)−(x+y)∣∣x+y∣≤ϵmach.\frac{ |(x \oplus y)-(x+y)| }{ |x+y| } \le \macheps.∣x+y∣∣(x⊕y)−(x+y)∣​≤ϵmach​.
(1.1.13)

Hence, the relative error in arithmetic is essentially the same as for the floating-point representation itself. However, playing by these rules can lead to disturbing results.

There are two ways to look at Example 1.1.4. On one hand, its two versions of the result differ by less than 1.2×10−161.2\times 10^{-16}1.2×10−16, which is very small — not just in everyday terms, but with respect to the operands, which are all close to 1 in absolute value. On the other hand, the difference is as large as the exact result itself! We formalize and generalize this observation in the next section. In the meantime, keep in mind that exactness cannot be taken for granted in floating-point computation.

Observation 1.1.1

We should not expect that two mathematically equivalent results will be equal when computed in floating point, only that they be relatively close together.
